# 简介

​	这是一个基于ImGui和ffmpeg的B站直播客户端，暂时仅支持windows平台（仅测试过WIN11，其他版本尚未测试）。

# 功能：

- 弹幕接收+渲染
- 弹幕发送 / +1
- 基于ImGui的DX12渲染
- 硬解支持 (D3D12VA)

# 使用说明

​	项目提供了Debug和Release版本的二进制文件直接下载，如果需要发送弹幕或者使用弹幕+1功能，请点击登录，并同时填写cookie的SESSDATA和bili_jct字段（请勿将含有 cookie 信息的截图或配置上传至公开平台）。每次正常退出程序，你的登录信息都会被自动保存，如果你不想使用自动保存功能，请在设置里修改。另外如果你想使用弹幕+1功能，只需要把鼠标放到你喜欢的弹幕上点击即可+1，但是为了防止无限制的+1，每次+1后有4秒的缓冲期。

# 编译

​	如果你想自己编译的话，请使用premake5生成vs工程，建议不要把工程放到移动硬盘或者机械硬盘，最好放到物理上的系统盘上，否则在使用vcpkg安装依赖时可能会导致一个时间相关问题让依赖编译失败。使用方法很简单，用premake5生成vs工程选择想要的编译配置，vs和vcpkg会自动处理依赖相关。

# 程序框架

​	目前所有代码架构仅为测试阶段，主要模块包括App、Network、UI、Renderer、Media、Event。

​	App:程序运行后，几乎所有逻辑都在App的run的while循环里面。由于ImGui立即渲染的特性，为了避免一些潜在的问题，在App运行的这一个循环出现的一些事件（例如打开直播间或者关闭直播间等），会在下一个循环的开头被处理。

​	Network:负责获取直播间信息和用户cookie等以及负责弹幕相关的网络服务（包括数据包的处理）

​	UI：所有需要被渲染的类都继承自RenderLayer，MainUI负责管理所有的RenderLayer，在App的循环中，MainUI会依次调用每个Layer的update和Render方法

​	Media：主要有四部分，PlayerbackController，AduioPlayer，VideoDecoder，AudioDecoder。PlayerbackController负责初始化ffmpeg信息和负责avpacket的生产，以及控制音视频的同步。AduioPalyer目前仅支持wasapi，VideoDecoder支持shader或者videoprocessor处理画面格式的转换。

​	Renderer：基于ImGui的渲染器，后端目前仅支持DX12。

​	Event：负责处理一些业务上的事件，比如app关闭或者房间的打开关闭。

# 可能存在的问题

​	目前对于可能存在的异常和错误处理的不是很到位，仅供娱乐使用，中途发生任何崩溃和错误都是可能的。另外，目前由于ffmpeg目前仅支持D3D12VA硬解，格式转换也用的DX12 api，所以你的系统版本和显卡需要支持DX12 api。另外，由于目前直播画面渲染和UI渲染都在同一个ImGui线程中，在尝试发送弹幕唤起中文输入法时导致的IME事件可能会让画面出现卡顿，而且在我的电脑上测试，ImGui有时不会将中文渲染到候选框。如果你实在接受不了这种情况或者默认的组合窗口，可以尝试在非全屏模式下点击其他窗口，比如程序运行弹出的CMD窗口，可能会让操作系统识别到聚焦窗口的变化而使用其他输入法。此外，建议不要开启垂直同步，目前发现开启垂直同步有可能会导致ffmpeg的解码出现问题导致画面花屏或者撕裂（目前原因不确定，只是猜测）。在选择视频格式转换时优先选择VideoProcessor模式，shader模式目前还在完善阶段，但是如果你接受不了VideoProcessor的高内存占用，选择shader模式大概率能将内存占用从200MB左右降低到150MB左右（以任务管理器为例）。另外，部分直播间弹幕服务器可能连接不上，后续会处理这个问题。
